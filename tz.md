# Техническое задание 
## Разработка веб-приложения для мониторинга физической активности и состояния здоровья

### 1. Общие сведения

**Цель проекта:**  
Разработка веб-приложения для мониторинга физической активности и состояния здоровья пользователя. Приложение должно собирать данные о физической активности пользователя, таких как количество шагов, сожженные калории, а также предоставлять персонализированные рекомендации по улучшению здоровья.

**Целевая аудитория:**  
Пользователи, желающие отслеживать и улучшать свои показатели физической активности и здоровья.

### 2. Функциональные требования

#### 2.1 Главная страница (Dashboard)

На главной странице отображаются основные показатели активности:
- Количество шагов за текущий день
- Сожженные калории
- График активности за неделю (линейный график с использованием Chart.js)
- Персонализированные рекомендации по улучшению физической активности и здоровья

**Элементы страницы:**
- Карточки с данными (шаги, калории)
- График с отображением активности
- Рекомендации по активности и здоровью
- Кнопки навигации для перехода на другие страницы (например, профиль, графики и т.д.)

#### 2.2 Страница мониторинга активности

На этой странице отображается более детализированная информация:
- Количество шагов за день
- Количество сожженных калорий
- Время, затраченное на физическую активность

**Элементы страницы:**
- График активности
- Кнопки для переключения между день, неделя, месяц

#### 2.3 Страница графиков (Stats)

На этой странице пользователю показываются различные графики:
- График шагов (по дням, неделям, месяцам)
- График сожженных калорий
- График изменения состояния здоровья (например, по результатам психологических опросников)

**Элементы страницы:**
- Линейные графики с использованием Chart.js
- Фильтры для выбора диапазона времени (день, неделя, месяц)

#### 2.4 Страница профиля пользователя

На этой странице пользователь может увидеть и редактировать свою личную информацию, а также данные о физической активности:
- Имя, возраст, пол
- История активности и здоровья
- История пройденных шагов, сожженных калорий, времени активности

**Элементы страницы:**
- Форма редактирования профиля (имя, возраст, пол)
- История активности и здоровья

#### 2.5 Страница опросников и оценки психологического состояния

На этой странице пользователь может пройти опрос для оценки своего психологического состояния:
- Оценка стресса
- Оценка уровня активности
- Оценка эмоционального состояния

**Элементы страницы:**
- Форма с вопросами и шкалами (от 1 до 5)
- Кнопка для отправки анкеты

#### 2.6 Страница входа/регистрации

Пользователь может войти в систему с помощью e-mail и пароля или через социальные сети (например, Google, Facebook).

**Элементы страницы:**
- Поле для ввода e-mail и пароля
- Кнопки для входа через соцсети (Google, Facebook)
- Кнопка для регистрации

### 3. Технические требования

#### 3.1 Используемые технологии

**Фронтенд:**
- React для построения UI
- Material UI для создания компонентов
- Chart.js для визуализации графиков
- Axios для взаимодействия с сервером и получения данных
- React Router для организации маршрутов

**Бэкенд** (по желанию, если планируется создание API):
- Node.js с использованием Express
- MongoDB или PostgreSQL для хранения данных пользователя и его активности
- API для получения данных о физической активности, состояниях пользователя и результатах опросников

#### 3.2 API
- `/api/activity`: Получение данных о физической активности пользователя (шаги, калории, активность)
- `/api/recommendations`: Получение персонализированных рекомендаций по здоровью
- `/api/user`: Получение и обновление данных пользователя
- `/api/psychological`: Получение и отправка результатов психологических опросников

#### 3.3 UI/UX
- Интерфейс должен быть интуитивно понятным, с простыми и понятными кнопками и подсказками
- Приложение должно быть отзывчивым, поддерживать мобильные устройства
- Использование цветовой схемы: голубые, зеленые и оранжевые оттенки для визуальной привлекательности
- Поддержка темной и светлой тем

### 4. План разработки

#### 4.1 Этап 1: Дизайн и проектирование
- Разработка макетов всех страниц приложения
- Проектирование структуры данных для хранения информации о пользователе и активности

#### 4.2 Этап 2: Разработка фронтенда
- Разработка главной страницы (Dashboard)
- Реализация страницы мониторинга активности
- Реализация страницы с графиками
- Реализация страницы профиля пользователя
- Разработка страницы для входа/регистрации

#### 4.3 Этап 3: Разработка бэкенда
- Создание API для работы с данными пользователя
- Интеграция с внешними сервисами для получения данных о физической активности (если нужно)
- Реализация логики для расчета шагов, калорий, BMR и TDEE

#### 4.4 Этап 4: Тестирование
- Тестирование всех функций и компонентов
- Тестирование безопасности приложения
